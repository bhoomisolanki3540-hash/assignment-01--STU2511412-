## Project Overview
This repository contains a comprehensive data engineering project covering Relational Databases, NoSQL, Data Warehousing, Vector Databases, and Data Lakehouse architectures. The project is divided into six specialized parts, culminating in an AI-powered architecture design for a hospital network.

## Repository Structure
* **part1-rdbms/**: 3NF Schema design and SQL normalization analysis.
* **part2-nosql/**: Product catalog design using MongoDB with ACID vs BASE analysis.
* **part3-datawarehouse/**: Star schema implementation and ETL transformation notes.
* **part4-vector-db/**: Python-based semantic similarity demo and legal tech reflection.
* **part5-datalake/**: Cross-format queries using DuckDB and Lakehouse recommendations.
* **part6-capstone/**: Hospital system architecture diagram and design justification.
* **datasets/**: Original raw data files used across all parts.
